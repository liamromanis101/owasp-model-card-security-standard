## 2. Intended Use

**Primary intended use:**  
Summarising internal business documents — meeting notes, reports, and support tickets — for ExampleCorp employees.

**Intended users:**  
ExampleCorp employees using the internal document management platform. Not intended for public-facing deployment.

**Out-of-scope uses:**  
- Medical, legal, or financial advice
- Customer-facing applications
- Processing of special category personal data as defined under GDPR Article 9
- Any use outside ExampleCorp's internal network without additional security review

### 5.1 Known Vulnerabilities

| ID | Description | Severity | Status | Reference |
|---|---|---|---|---|
| EC-2024-001 | Model can be induced to reproduce training data fragments via repeated token prompting | Low | Mitigated — output length cap and repetition penalty applied at inference | Internal advisory EC-2024-001 |

### 5.2 Adversarial Testing

| Field | Detail |
|---|---|
| Adversarial testing performed | [x] Yes |
| Testing methodology | Internal red team exercise; prompt injection testing using OWASP LLM Top 10 as test framework |
| Scope of testing | Prompt injection, jailbreaking, indirect prompt injection via document content, data extraction attempts |
| Results summary | No critical findings. One low-severity memorisation issue identified and mitigated (see EC-2024-001 above). Model showed resistance to common jailbreak patterns. |
| Full report location | Internal: confluence.examplecorp.example/ai-security/redteam-v2.1 |

### 5.3 Output Filtering & Safety Controls

| Field | Detail |
|---|---|
| Output filtering in place | [x] Yes |
| Filtering method | Regex-based PII detection on outputs (Presidio); profanity filter; max output length 1,024 tokens |
| Known bypass risks | Determined adversary with access to the system prompt could potentially circumvent output filters via indirect injection through document content. Mitigated by input sanitisation at ingestion. |

## 11. Ethical Considerations

**Bias assessment performed:** [x] Yes  
**Summary of findings:** Model showed mild preference for formal register in summaries regardless of source document tone. No demographic bias detected in available test sets.

**Fairness considerations:** Test sets were English-only. Performance on documents with non-standard English (regional dialects, non-native written English) has not been formally assessed.

**Environmental impact:** Training produced an estimated 1.2 tonnes CO₂e based on hardware energy consumption. Inference runs on shared infrastructure; per-query footprint not separately measured.
